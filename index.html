Beyond the User.
The Human-Agent Pair.

The fundamental unit of analysis has shifted. We no longer design for users; we design for collaborative entities consisting of a human and an AI agent working together, evolving over time.

Explore Definitions View Artifacts

Evolution of Artifacts

Traditional

User Stories

Best suited for conventional apps without agent collaboration.

// Format "As a [user],
I want [goal]
so that [reason]."

Key Components

  • Role
  • Goal
  • Reason
Automation Era

Agent Stories

For autonomous agents or background automation independent of deep collaboration.

Format Structure

  • Current Capabilities What the agent can do today.
  • Future Capabilities Post-maturity abilities.
  • Triggers Activation conditions.
Example
Current: Categorize tickets by urgency, suggest templates.
Future: Detect sentiment trends, proactive outreach.
Triggers: New ticket, sentiment drop, 24h no response.
Collaborative Era

The HAP Plan

The primary requirements artifact for collaborative systems. Maps how the pair works together, builds trust, and evolves.

Core Sections

  • Pair Goal & Setup Shared KPIs, role allocation, tooling.
  • Gates & Guardrails Confidence thresholds, safety boundaries.
  • Dynamics & Evolution Feedback loops, skill drifts, metrics.

Key Differentiators

  • Models evolving relationship over time
  • Defines trust-building milestones
  • Captures emergent pair capabilities
  • Balances autonomy with accountability
