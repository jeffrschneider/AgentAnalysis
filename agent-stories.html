Automation Era

Agent Stories

Requirements artifacts for autonomous agents and background automation systems.

The Approach

Agent Stories extend traditional user stories to capture the behavior of autonomous AI agents. Unlike user stories that focus on human-initiated actions, Agent Stories describe what an AI system does independently—often in the background, without direct human involvement.

This artifact introduces the concept of temporal capability: distinguishing between what the agent can do today versus what it will be able to do as it matures. Agent Stories also formalize triggers—the conditions or events that cause the agent to act autonomously.

Agent Stories are ideal for automation systems, background processors, monitoring agents, and any AI that operates with minimal human oversight. They capture the agent as an independent actor, though they don't fully address collaborative dynamics where human and agent work as partners.

Artifact Outline

Current Capabilities

What the agent can reliably do today with existing training, data, and integrations.

Future Capabilities

Abilities expected post-maturity: after more training, data accumulation, or system upgrades.

Triggers

Events, conditions, or thresholds that activate the agent's autonomous behavior.

Standard Format

AGENT: [Agent name/type]
CURRENT: [What it can do now]
FUTURE: [What it will do post-maturity]
TRIGGERS: [Activation conditions]
CONSTRAINTS: [Boundaries and limitations]

Optional Components

  • Fallback Behavior: What happens when the agent cannot complete its task
  • Escalation Path: When and how to involve human oversight
  • Success Metrics: How agent performance is measured
  • Learning Objectives: What the agent should improve over time

Example

Agent Story
AGENT: Support Ticket Triage Agent
CURRENT:
- Categorize incoming tickets by type (billing, technical, general) - Assign urgency scores based on keyword analysis - Suggest response templates from knowledge base - Route tickets to appropriate team queues
FUTURE:
- Detect customer sentiment trends across ticket volume - Predict ticket escalation likelihood - Initiate proactive outreach for at-risk accounts - Auto-resolve routine issues without human review
TRIGGERS:
- New ticket submitted to support system - Sentiment score drops below threshold - Ticket age exceeds 24 hours without response - Customer marked as VIP or enterprise tier
CONSTRAINTS:
- Cannot access customer payment information - Must escalate legal/compliance issues to human - Cannot modify account status or issue refunds

Why This Works for Automation

This Agent Story captures an autonomous system that acts independently based on defined triggers. The temporal split between current and future capabilities enables incremental rollout—the agent starts with safe categorization tasks and graduates to more autonomous actions as it proves reliability. The constraints establish hard boundaries that don't require human-in-the-loop for every action.

When to Use Agent Stories

Ideal For

  • Background processing systems
  • Monitoring and alerting agents
  • Data pipeline automation
  • Scheduled task execution
  • Event-driven workflows

Not Ideal For

  • Deep human-agent collaboration
  • Trust-building over time
  • Dynamic responsibility allocation
  • Emergent pair capabilities
  • Human judgment integration

For collaborative systems where humans and agents work as partners, consider the HAP Plan.
